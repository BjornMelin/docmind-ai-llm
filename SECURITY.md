# Security policy

## Supported versions

Security fixes target the latest release in the current major line. Pre-1.0
releases are unsupported after 1.0.0 is published.

## Report a vulnerability

Do not disclose vulnerability details in a public issue. Use GitHub's
[private vulnerability report](https://github.com/BjornMelin/docmind-ai-llm/security/advisories/new).
If that form is unavailable, open a public issue that asks the maintainer for a
private contact channel without including technical details.

Include the affected version, impact, reproduction conditions, and a minimal
proof of concept. Do not attach real documents, credentials, tokens, private
URLs, personal data, or production logs.

Reports are handled on a best-effort basis. There is no guaranteed response or
remediation time. The repository threat model is documented in
[SPEC-011](docs/specs/spec-011-security-privacy.md).
