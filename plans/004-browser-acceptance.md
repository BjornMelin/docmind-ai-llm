## Why this matters

Streamlit AppTest proves widget semantics but cannot detect focus order,
keyboard operation, sidebar/mobile navigation, overflow, contrast, Plotly
sizing, browser console errors, or visual states. A compact browser lane is the
prerequisite for the redesign plans.

## Exact launch contract

In `playwright.config.ts`, use Playwright `webServer` to run:

```text
uv run streamlit run app.py --server.headless=true --server.address=127.0.0.1 --server.port=8510
```

Use base URL `http://127.0.0.1:8510`, `reuseExistingServer: false` in CI, and
pass this complete `webServer.env` map so a repo-root `.env` cannot select a
credentialed or remote backend:

```text
DOCMIND_CONFIG__DOTENV_PRIORITY=env_first
DOCMIND_DATA_DIR=.browser-test-data/data
DOCMIND_CHAT__SQLITE_PATH=.browser-test-data/data/chat.db
DOCMIND_PARSING__MODEL_CACHE_DIR=.browser-test-data/models
DOCMIND_LLM_BACKEND=ollama
DOCMIND_LLM_REQUEST__MODEL=browser-test-model
DOCMIND_OLLAMA_BASE_URL=http://127.0.0.1:9
DOCMIND_DATABASE__QDRANT_URL=http://127.0.0.1:9
DOCMIND_SECURITY__ALLOW_REMOTE_ENDPOINTS=false
DOCMIND_SECURITY__ENDPOINT_ALLOWLIST=["http://127.0.0.1"]
DOCMIND_ENABLE_GPU_ACCELERATION=false
DOCMIND_ANALYTICS_ENABLED=true
HF_HUB_OFFLINE=1
TRANSFORMERS_OFFLINE=1
HF_HOME=.browser-test-data/hf
XDG_CACHE_HOME=.browser-test-data/cache
```

Resolve all relative values against the repository root. The exact package
scripts recreate `.browser-test-data` before the web server starts. Do not add
production fallbacks or test flags. Tests
must not call an external or credentialed model. They may submit one request to
a guaranteed-loopback closed port to assert sanitized provider failure; Plan
002's success semantics remain covered by AppTest until a credential-free
public service seam exists.

## CI workflow

`browser-ui.yml` must use `actions/checkout@v7`, Python `3.12.13`, the same
pinned `astral-sh/setup-uv` action/version as `ci.yml`, and
`oven-sh/setup-bun@v2` with Bun `1.3.14`. Run:

```bash

```

Set `timeout-minutes: 15`, permissions `contents: read`, concurrency cancellation
matching `ci.yml`, and upload `playwright-report/` plus `test-results/` only on
failure with seven-day retention.

## Acceptance cases

Implement role/name assertions, not broad pixel snapshots:

1. Desktop 1440×1000: app loads, all four navigation destinations open, and no
   page has horizontal document overflow.
2. Mobile 390×844: navigation remains reachable and page content does not
   overflow horizontally.
3. Documents empty state and mutation controls render; Plan 001 active-job
   disablement is asserted using only state reachable without production
   bypasses. If active state cannot be reached in a real browser without a
   production hook, leave that behavior in AppTest and record the boundary.
4. Settings validation plus the current `Apply runtime` and `Save` controls
   render without writing `.env`; assert behavior/lifetimes, not the future
   labels from Plan 008. Plan 008 must update these browser assertions when it
   introduces its exact explanatory copy.
5. Chat renders Plan 003's honest embedding-unavailable/degraded state from the
   isolated empty offline cache. If a loopback-only provider failure remains
   reachable without model artifacts, assert its sanitization too; do not
   require a successful model call.
6. Analytics enabled empty state renders using the explicit environment above;
   its disabled state remains covered by AppTest because one webServer has one
   immutable environment.
7. Each stable page runs axe; fail on serious or critical violations.
8. Fail on uncaught page errors or console messages of type `error`, with a
   small documented allowlist only for proven Streamlit framework noise.
